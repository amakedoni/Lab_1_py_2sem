# Лабораторная работа №1 — Источники задач и контракты

## Цель работы

Освоить **duck typing** и **контрактное программирование** на примере источников задач.

Предметная область: абстрактная платформа обработки задач, которая умеет принимать задачи из различных источников через единый поведенческий контракт.


## Описание компонентов

### `Task` (models.py)
Dataclass с двумя полями:
- `id: str` — уникальный идентификатор задачи
- `payload: Any` — произвольные данные

### `TaskSource` (contracts.py)
`runtime_checkable` Protocol, определяющий единственный метод:
```python
def get_task(self) -> Iterable[Task]: ...
```

### Источники задач (sources.py)

| Класс | Описание |
|---|---|
| `FileTaskSource` | Загружает задачи из JSON-файла |
| `GeneratorTaskSource` | Принимает программный генератор словарей |
| `ApiStubTaskSource` | Имитирует получение задач из внешнего API |

Все источники реализуют контракт `TaskSource` **без общего базового класса** — только через duck typing.

### `TaskLoader` (loader.py)
Принимает любой объект, реализующий `TaskSource`, и валидирует его через `validate_source()`. Метод `load_all()` возвращает `list[Task]`.

### `cli.py` — Интерактивный режим
CLI-интерфейс для ручного тестирования всех трёх источников задач прямо из терминала. Поддерживает выбор источника, ввод данных и вывод загруженных задач с обработкой ошибок.

## Функциональные требования

- Все источники реализуют единый контракт через `typing.Protocol`
- Запрещено использование общего базового класса
- Runtime-проверка контракта через `isinstance(source, TaskSource)`
- Архитектура открыта для расширения без изменения существующего кода
- Аннотации типов для всех публичных интерфейсов

## Запуск

### Интерактивный режим (CLI)

```bash
python main.py
```

Откроется меню выбора источника задач:

```
╔══════════════════════════════════╗
║   Task Processing Platform CLI   ║
╚══════════════════════════════════╝

┌─ МЕНЮ ───────────────────────────
│  1. 📄 Из файла (JSON)
│  2. ⚙️  Генератор (ввод JSON)
│  3. 🌐 API-заглушка (ввод JSON)
│  q. Выход
└──────────────────────────────────
Выбери источник:
```

#### Примеры ввода

**Режим 1 — Из файла:**

Создай файл `tasks.json`:
```json
[
  {"id": "1", "payload": "Обработать заказ #101"},
  {"id": "2", "payload": "Отправить уведомление пользователю"},
  {"id": "3", "payload": "Пересчитать статистику"}
]
```
Затем введи путь к файлу:
```
Выбери источник: 1
  Путь к JSON-файлу: tasks.json
```

**Режим 2 — Генератор:**
```
Выбери источник: 2
  Введи JSON-массив задач: [{"id": "10", "payload": "Проверить состояние ресурса"}, {"id": "11", "payload": "Запустить фоновую задачу"}]
```

**Режим 3 — API-заглушка:**
```
Выбери источник: 3
  Введи JSON-массив задач для API-заглушки: [{"id": "api-001", "payload": "Обработать входящие данные"}, {"id": "api-002", "payload": "Синхронизировать базу"}]
```

#### Пример вывода
```
✅ Загружено задач: 2
  [api-001] Обработать входящие данные
  [api-002] Синхронизировать базу
```

#### Обработка ошибок

| Ситуация | Сообщение CLI |
|---|---|
| Файл не найден | `❌ Файл не найден: tasks.json` |
| Невалидный JSON (одинарные кавычки и т.д.) | `❌ Невалидный JSON, попробуй снова.` |
| Объект не реализует контракт | `❌ Ошибка контракта: Объект ... не реализует протокол TaskSource` |
| Отсутствует поле `id` или `payload` | `❌ Неожиданная ошибка: KeyError` |

> ⚠️ В JSON обязательно используй **двойные кавычки**: `"id"`, а не `'id'`.
